# 데모 신청 설문조사 관리 문서

> AIR 랜딩페이지의 데모 신청 설문조사 항목 및 관리 가이드

---

## 설문조사 개요

**목적**: 데모 신청 시 필요한 기본 정보 수집 및 고객 맞춤형 상담 준비  
**페이지 경로**: `/demo-request`  
**제출 후**: 감사 메시지 표시 및 이메일 알림 (선택적)

---

## 필수 입력 항목 (Required Fields)

### 1. 이름 (Name)
- **필드명**: `name`
- **타입**: `text`
- **검증 규칙**:
  - 필수 입력
  - 최소 2자, 최대 50자
  - 한글, 영문, 공백만 허용
- **플레이스홀더**: "이름을 입력해주세요"
- **에러 메시지**: "이름을 올바르게 입력해주세요 (2-50자)"

### 2. 이메일 (Email)
- **필드명**: `email`
- **타입**: `email`
- **검증 규칙**:
  - 필수 입력
  - 이메일 형식 검증
  - 도메인 검증 (선택적)
- **플레이스홀더**: "example@company.com"
- **에러 메시지**: "올바른 이메일 주소를 입력해주세요"

### 3. 휴대폰번호 (Phone)
- **필드명**: `phone`
- **타입**: `tel`
- **검증 규칙**:
  - 필수 입력
  - 한국 휴대폰 번호 형식: `010-XXXX-XXXX`
  - 숫자만 입력 후 자동 포맷팅
- **플레이스홀더**: "010-0000-0000"
- **에러 메시지**: "올바른 휴대폰 번호를 입력해주세요"

---

## 선택 입력 항목 (Optional Fields)

### 4. 회사명 (Company)
- **필드명**: `company`
- **타입**: `text`
- **검증 규칙**:
  - 선택 입력
  - 최대 100자
- **플레이스홀더**: "회사명을 입력해주세요 (선택)"
- **에러 메시지**: 없음 (선택 항목)

### 5. 부서 (Department)
- **필드명**: `department`
- **타입**: `text`
- **검증 규칙**:
  - 선택 입력
  - 최대 50자
- **플레이스홀더**: "부서명을 입력해주세요 (선택)"
- **에러 메시지**: 없음 (선택 항목)

### 6. 문의사항 (Inquiry)
- **필드명**: `inquiry`
- **타입**: `textarea`
- **검증 규칙**:
  - 선택 입력
  - 최대 500자
  - 줄바꿈 허용
- **플레이스홀더**: "문의사항을 입력해주세요 (선택)"
- **에러 메시지**: 없음 (선택 항목)

## 폼 레이아웃

### 디자인 원칙
- **Toss 스타일**: 테두리 없는 입력창, 배경색으로 구분
- **레이블**: 입력창 위에 굵은 질문 형태로 배치
- **간격**: 입력창 간 충분한 여백 (`gap-6`)
- **버튼**: Primary 버튼, 전체 너비 (모바일) 또는 적절한 너비 (PC)

### 레이아웃 구조
```
[제목: 무료 데모 신청]
[설명: AIR의 강력한 기능을 직접 체험해보세요]

[이름 입력 필드]
[이메일 입력 필드]
[휴대폰번호 입력 필드]
[회사명 입력 필드] (선택)
[부서 입력 필드] (선택)
[문의사항 텍스트 영역] (선택)

[제출 버튼]
```

## 폼 검증 로직

### 클라이언트 사이드 검증
- **라이브러리**: React Hook Form + Zod
- **검증 시점**: 
  - 실시간 검증 (onChange)
  - 제출 시 최종 검증 (onSubmit)
- **에러 표시**: 필드 하단에 빨간색 텍스트로 표시

### 서버 사이드 검증 (향후 구현)
- API 엔드포인트에서 재검증
- 스팸 방지 (reCAPTCHA 또는 hCaptcha)
- 중복 제출 방지

---

## 제출 처리

### 제출 성공 시
1. 감사 메시지 페이지로 이동 (`/demo-request/thank-you`)
2. 입력 데이터는 서버로 전송 (API 호출)
3. 이메일 알림 발송 (선택적)

### 감사 메시지 페이지
- **제목**: "데모 신청이 완료되었습니다"
- **내용**: 
  - "곧 담당자가 연락드리겠습니다"
  - "일반적으로 1-2일 내에 연락드립니다"
- **액션**: "홈으로 돌아가기" 버튼

## 데이터 저장 (향후 구현)

### 저장 형식
```json
{
  "id": "unique-id",
  "name": "홍길동",
  "email": "hong@example.com",
  "phone": "010-1234-5678",
  "company": "ABC 기업",
  "department": "안전관리팀",
  "inquiry": "문의사항 내용",
  "submittedAt": "2024-01-XXT00:00:00Z",
  "status": "pending"
}
```

### 저장 위치
- 데이터베이스 (향후 구현)
- 또는 이메일로 전송 (초기 단계)
